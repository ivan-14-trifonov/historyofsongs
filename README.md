# Музыкальная литература МСЦ ЕХБ

База знаний по музыкальной литературе.

## Технологии

### Фреймворк и язык

- **[Next.js 14](https://nextjs.org/)** — React-фреймворк для серверного рендеринга (SSR) и статической генерации (SSG)
- **[TypeScript 5](https://www.typescriptlang.org/)** — типизированный язык программирования
- **[React 18](https://react.dev/)** — библиотека для построения пользовательских интерфейсов

### Работа с контентом

- **[react-markdown](https://github.com/remarkjs/react-markdown)** — рендеринг Markdown-контента в React-компоненты
- **[remark-gfm](https://github.com/remarkjs/remark-gfm)** — плагин для поддержки GitHub Flavored Markdown
- **[gray-matter](https://github.com/jonschlinkert/gray-matter)** — парсинг front-matter из Markdown-файлов (метаданные: заголовок, порядок и т.д.)

### Развёртывание

- **[Vercel](https://vercel.com/)** — платформа для хостинга и деплоя (конфигурация в `vercel.json`)

### Инструменты

- **[ESLint](https://eslint.org/)** — линтинг кода
- **[TSC](https://www.typescriptlang.org/)** — проверка типов

## Архитектура

### Источники данных

Контент хранится в виде Markdown-файлов в директории `content/`. Каждый файл может содержать front-matter с метаданными:

```markdown
---
title: Заголовок статьи
---

Текст статьи...
```

### Маршрутизация

- `/` — главная страница со списком разделов
- `/folder/[...slug]` — просмотр содержимого папок (вложенная структура)
- `/content/[...slug]` — отображение отдельных статей

### Статическая генерация

Проект использует `generateStaticParams()` для предварительной генерации всех страниц на этапе сборки (`next build`). Это обеспечивает быструю загрузку и возможность деплоя на статические хостинги.

## Контент

Markdown-файлы хранятся в отдельном репозитории:

- **Репозиторий с контентом:** https://github.com/ivan-14-trifonov/songs_info
- **Локальная папка:** `content/`

### Загрузка контента локально

Контент не хранится в этом репозитории — он подтягивается из внешнего репозитория. Папка `content/` добавлена в `.gitignore`, поэтому Git её игнорирует. Для локальной работы нужно загрузить контент вручную:

```bash
# Клонировать контент в папку content/
git clone --depth 1 https://github.com/ivan-14-trifonov/songs_info.git content

# Если нужно редактировать контент и пушить изменения:
git clone https://github.com/ivan-14-trifonov/songs_info.git content
```

Для обновления контента до последнего коммита:

```bash
cd content
git pull origin main
```

> **Важно:** Изменения в контенте коммитятся в репозиторий `songs_info`, а не в основной. Для редактирования контента работайте в папке `content/`:
> ```bash
> cd content
> git add .
> git commit -m "..."
> git push
> ```

### Автоматическая загрузка на Vercel

При каждом деплое Vercel автоматически клонирует контент-репозиторий перед сборкой. Это реализовано в `package.json`:

```json
"build": "rm -rf content && git clone --depth 1 https://github.com/ivan-14-trifonov/songs_info.git content && rm -rf content/.git && next build"
```

Порядок действий:
1. `rm -rf content` — удаляет старую папку (если осталась с предыдущего билда)
2. `git clone --depth 1 ...` — клонирует последний коммит контент-репозитория (без полной истории для скорости)
3. `rm -rf content/.git` — удаляет `.git`, чтобы внутренности репозитория не попали в сборку
4. `next build` — запускает сборку Next.js

Таким образом на сайте всегда актуальная версия контента без необходимости вручную обновлять субмодули.

## Запуск

```bash

npm install

# Запуск в режиме разработки

# Сборка для продакшена
npm run build

# Запуск собранной версии
npm run start
```
