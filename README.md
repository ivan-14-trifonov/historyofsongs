# Музыкальная литература МСЦ ЕХБ

База знаний по музыкальной литературе.

## Технологии

### Фреймворк и язык

- **[Next.js 14](https://nextjs.org/)** — React-фреймворк для серверного рендеринга (SSR) и статической генерации (SSG)
- **[TypeScript 5](https://www.typescriptlang.org/)** — типизированный язык программирования
- **[React 18](https://react.dev/)** — библиотека для построения пользовательских интерфейсов

### Работа с контентом

- **[react-markdown](https://github.com/remarkjs/react-markdown)** — рендеринг Markdown-контента в React-компоненты
- **[remark-gfm](https://github.com/remarkjs/remark-gfm)** — плагин для поддержки GitHub Flavored Markdown
- **[gray-matter](https://github.com/jonschlinkert/gray-matter)** — парсинг front-matter из Markdown-файлов (метаданные: заголовок, порядок и т.д.)

### Развёртывание

- **[Vercel](https://vercel.com/)** — платформа для хостинга и деплоя (конфигурация в `vercel.json`)

### Инструменты

- **[ESLint](https://eslint.org/)** — линтинг кода
- **[TSC](https://www.typescriptlang.org/)** — проверка типов

## Структура проекта

```
├── app/                    # Next.js App Router (страницы и лейауты)

│   ├── Breadcrumb.tsx      # Навигационная цепочка (хлебные крошки)

└── public/                 # Статические файлы
```

## Архитектура

### Источники данных

Контент хранится в виде Markdown-файлов в директории `content/`. Каждый файл может содержать front-matter с метаданными:

```markdown
---
title: Заголовок статьи
---

Текст статьи...
```

### Маршрутизация

- `/` — главная страница со списком разделов
- `/folder/[...slug]` — просмотр содержимого папок (вложенная структура)
- `/content/[...slug]` — отображение отдельных статей

### Статическая генерация

Проект использует `generateStaticParams()` для предварительной генерации всех страниц на этапе сборки (`next build`). Это обеспечивает быструю загрузку и возможность деплоя на статические хостинги.

## Контент

Markdown-файлы хранятся в отдельном репозитории и подключены как **git submodule**:

- **Репозиторий с контентом:** https://github.com/ivan-14-trifonov/songs_info
- **Локальная папка:** `content/`

### Клонирование

При клонировании проекта **обязательно** инициализируйте сабмодули:

```bash
# Клонирование с сабмодулями (рекомендуется)
git clone --recurse-submodules <url-репозитория-кода>

# Если уже склонировали без сабмодулей
git submodule update --init --recursive
```

### Обновление контента

```bash
# Обновить контент из удалённого репозитория
git submodule update --remote content

# Закоммитить новую версию контента
git add content
git commit -m "update: обновить контент"
```

> **Важно:** Изменения в контенте коммитятся в репозиторий `songs_info`, а не в основной. Для редактирования контента можно работать напрямую в папке `content/`:
> ```bash
> cd content
> git add .
> git commit -m "..."
> git push
> ```
